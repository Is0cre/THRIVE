# Session 2026-04-03

## What was decided

### Architecture
- Three tier progression system — Regulate, Stabilise, Process
- Ambient unlocking — no announcements, no progress bars
- Non-linear design — a space, not a course
- Panic button always visible, one tap, no decisions required
- Window of Tolerance check-in gates tool access per session

### Clinical safety
- Wim Hof hard gates — 8 confirmation gates, no opt-out, every time
- Bilateral stimulation never called EMDR in UI
- Bilateral stimulation labelled as between-session support, not therapy
- Clinical comment blocks required on all safety-critical functions
- Marit Tandberg named as clinical advisor in CLAUDE.md

### Local LLM
- Explore on-device small model (Gemma 2B / Phi-3 Mini / Llama 3.2 3B)
- MediaPipe LLM Inference as integration path
- Used for check-in routing and journaling companion only
- Never a general chatbot
- System prompt written by Marit defines clinical behavior

### The vault (deferred — post beta)
- Patient-owned encrypted health data
- QR share in person, encrypted blob relay remote
- Doctor consent gate — timestamped, signed, stored as patient receipt
- Relay sees only encrypted bytes — nothing to subpoena
- Self-hostable Go microservice
